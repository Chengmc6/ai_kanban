# AI Kanban

## プロジェクト概要

AI Kanbanは、AIを活用したタスク管理・プロジェクト管理システムです。ユーザーがより効率的にプロジェクトを管理し、タスクを追跡できるように設計されています。

勉強のために開発しています。

## 学習過程と設計思考

このプロジェクトは、実務に近い開発経験を積むことを目的として、
バックエンド・AI処理層・フロントエンドを分離した構成で設計しました。
前職テストエンジニアで得た経験をもとに、「現場ではどのようにシステムが作られているのか」を意識しながら進めています。

### 1. 学習方法について（AI を活用した学習スタイル）

私は AI を「コードを代わりに書く存在」ではなく、  
**思考を補助し、設計や実装を検証するパートナー**として活用しています。

- まず自分で処理の流れや責務（Controller / Service / Repository）を整理
- API 構成やデータの流れを考えた上で AI に相談
- 提案された実装について  
  「なぜこの書き方なのか」「このプロジェクトに適しているか」  
  を確認し、理解できたもののみを採用

学習段階では IDE の自動補完に過度に依存せず、  
**自分で書く → エラーを読む → 修正する**  
というサイクルを重視しています。

---

### 2. バックエンド設計の考え方（Java / Spring Boot）

バックエンドでは、責務分離と拡張性を意識した設計を行っています。

- Controller：リクエスト受付とレスポンス制御
- Service：業務ロジックの集約
- Mapper / Repository：データアクセスの分離

特に以下の点を重視しました。

- 認証・認可（Spring Security + JWT）を  
  業務ロジックから分離すること
- 将来的な機能追加を想定し、  
  クラスやメソッドの責務が肥大化しない構造にすること
- API のレスポンス形式・エラーコードを統一し、  
  フロントエンドと連携しやすい設計にすること

---

### 3. AI レイヤー設計の考え方（Python / FastAPI）

AI Kanban プロジェクトでは、  
**AI 専用レイヤーを Java バックエンドとは分離**して設計しています。

- Java：業務データ管理・認証・永続化
- AI レイヤー（FastAPI）：
  - プロンプト処理
  - AI との通信
  - レスポンスの整形

この構成により、

- AI 技術の変更・拡張が容易
- バックエンドの責務が明確
- 将来的なマイクロサービス化が可能

といったメリットがあると考えています。

FastAPI や AI 周辺技術は未経験からのスタートでしたが、  
**実際のプロジェクトとして実装することで、全体の流れと役割の理解を優先**しました。

---

### 4. フロントエンド設計の考え方（Vue 3 / Nuxt）

フロントエンドでは以下を意識しています。

- 状態管理（Pinia）と UI ロジックの分離
- API 通信処理の共通化
- コンポーネント単位での再利用性

文法理解だけでなく、  
**「実際に動く画面を作ること」**を重視して学習を進めています。

---

### 5. 資格よりも「説明できる実装」を重視

資格取得も重要だと考えていますが、  
現時点では **自分が書いたコードを説明できること**を最優先にしています。

- なぜこの技術を選んだのか
- なぜこの構造にしたのか
- どこが難しかったか

これらを自分の言葉で説明できる状態を目標に、  
プロジェクトベースで学習を進めています。

---

### 6. 今後の課題と方針

- FastAPI / Python の基礎理解をさらに深める
- AI 周辺技術（LangChain 等）の役割と適切な使い所の整理
- フロントエンドの実装量を増やし、UI の完成度を向上させる

短期間で表面的に覚えるのではなく、  
**「理解 → 実装 → 振り返り」**のサイクルを回しながら成長していきたいと考えています。


## 主な機能

- **プロジェクト管理**: プロジェクトの作成、更新、削除、メンバー管理
- **タスク管理**: タスクの作成・管理・ステータス追跡
- **AI統合**: AIを活用した自動タスク生成・最適化機能
- **ユーザー認証**: ユーザー登録・ログイン・権限管理
- **Kanbanボード**: 視覚的なカンバンボード表示

## インストール・セットアップ

### 要件

- Java 17以上
- MySQL 8.0以上
- Python 3.8以上
- Maven 3.6以上

### バックエンド設定

1. MySQLデータベースを作成
```sql
CREATE DATABASE ai_kanban;
```

2. `backend/src/main/resources/application.properties`でデータベース接続情報を設定

3. バックエンドを起動
```bash
cd backend
mvn spring-boot:run
```

### AI層設定

1. Python環境をセットアップ
```bash
cd ai-layer
python -m venv venv
source venv/bin/activate  # Linux/Mac
# または
venv\Scripts\activate  # Windows
```

2. 依存パッケージをインストール
```bash
pip install -r requirements.txt
```

3. AI層を起動
```bash
python app/main.py
```
